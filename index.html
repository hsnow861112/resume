구현 예제

2. 실시간 요청이 서버(가운데 원통) 영역으로 들어 갔을때 버블 효과.

문제 해결 사례

- 스피드바 차트 애니매이션 구현.

[문제 설명]
APM 대시보드의 상징적인 차트인 스피드바 차트를 구현했어야 했는데 여러 차트 라이브러리에 대해서 기술 조사를 해봤지만 어디서도 해당 차트를 제공하는 곳을 찾지 못해서 직접 구현해야 했었음.

[해결 과정]
해당 차트를 구현했을때 어려웠던 점은 2개 있었음. 첫번째는 RPS, TPS를 나타내는 선이 원통영역으로 날라갈때 커브 애니메이션의 구현 이었음. 여러 생각은 해본 결과 아래와 같은 방법으로 구현 할 수 있었음.

1. 1번 직선의 첫번째 점 x1을 점점 증가시키면서 직선을 왼쪽에서 오른쪽으로 이동시킴.

2. 이동 도중 그려야 하는 직선의 길이(width) 안에 고정된 커브 영역 좌표 x2가 포함되어있으면 직선을 x1 ~ x2까지만 그림.

3. 커브 애니메이션을 구현하기 위해 고정된 커브 영역좌표에서 선의 마지막 좌표로 향하는 직선을 그림.

4. 2번 직선의 우측 마지막 x좌표는 x2 + (직선의 길이(width) – (x2 – x1)).

5. y값을 구하기 위해 선의 마지막 좌표와 고정된 커브 영역 좌표 두개를 활용해서 기울기를 구하고 ((y3 – y2) / (x3 – x2)) 일차함수 y = ax + b 에서 a에 기울기를 대입 하고 x에 2번 직선의 우측 마지막 x좌표를 대입 해서 b의 값을 구해서 방정식을 완성 시킨 후 y값을 구한 후 고정된 커브 영역 좌표와 2번 직선의 우측 마지막 x좌표를 연결 해서 선을 그림.

[해결 과정]
두번째 선이 원통영역으로 다 날라가고 나서 물방울이 터지는 애니매이션 효과 였고 아래와 같은 방법으로 구현 할 수 있었음.

1. 한 점에 있는 여러 개의 원이 퍼지는 애니매이션을 구현하기 위해서 각 각도에 따른 좌표를 구해야 했음.

2. 전방위로 퍼져나가야 하기 때문에 360도에서 원의 개수를 나누어서 각도(DEGREE)를 구하고 원이 퍼져 나갈 마지막 길이(R)를 정의함.

3. 길이(R)과 각도(DEGREE)를 안다면 삼각함수를 사용해서 SIN(degree) = Y / R, COS(DEGREE) = X / R을 구할 수 있고 그에 따라 Y = SIN(DEGREE) * R, X = COS(DEGREE) * R을 유추 할 수 있음.

4. 길이(R)을 0에서 점점 증가시킨다음 그 상태에서 X, Y값에 원을 그려서 점점 퍼져 나가는 애니매이션을 구현함. 퍼저나가는 원의 개수 및 크기를 랜덤화 시켜서 좀 더 극적인 효과가 보여지게끔 구현함.

미리 형태가 정해져 있는 대시보드가 아닌 마치 파워 포인트 와 같이 성능 항목들을 drag & drop으로 사용자가 원하는 위치에 설정하고 또한 위치 시킨 컴포넌트들의 width, height 도 drag 로 설정 할 수 있게 설계 하였고 이러한 기능들을 구현하기 위해서 jQuery UI의 Draggable, Droppable, Resizable을 기술 조사하여 개발함.

로그의 before, after를 비교해서 하이라이팅 표시를 할때 diff.js 라이브러리를 사용했는데 그 때 JSON 로그 데이터가 커지면 커질 수록 diff.js에서 성능이 현저하게 느려져 화면이 멈추는 현상이 나타남.

[해결 과정]
해당 data를 메인쓰레드에서 처리 하지 않고 WebWorker에 보내서 처리 하기로 하였지만 diff.js라는 라이브러리를 사용해야 했음. WebWorker 안에서는 메인 쓰레드에 import 되어있는 diff.js를 사용 할 수 없기 때문에 따로 script를 import 할 수 있는 방법을 찾아야 했고 https://developer.mozilla.org/en-US/docs/Web/API/WorkerGlobalScope/importScripts 해당 문서에서 그 방법을 찾아 해당 라이브러리를 사용할 수 있었음. WebWorker는 기본적으로 외부 js에 onmessage를 구현해야 하는데 유지 보수할 때 Vue 프로젝트와 Web Application에 있는 js를 번갈아 가면서 보면 가독성이 떨어지기 때문에 https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API/Using_web_workers 참조해서 Embedded형태의 WebWorker를 사용해서 가독성을 좋게했음.
