iManager

연계 시스템에 필요한 데이터를 설정, 모니터링 하기 위한 Web Application
- 사용 기술
* JavaScript, Vue.js
- 인력 구성
* 개발자: 3명 , 디자인: 1 명
- 상세 업무
1. JSP 에서 Vue로 개발 방식 변경

- 프론트 엔드 기술 발전에 따라 레거시 jsp 대신에 SPA(Vue, React, Angular) 방식을 채택 하기로함.

- Vue, React, Angular 중에 기술 조사 후 Vue를 선택.

- iManager는 주로 설정하는 하는 웹 어플리케이션이라서 비교적 간단한 화면들이 많고 사업 특성상 백엔드 개발자들이 화면을 만들기도 해서 러닝 커브가 작은 Vue를 선택하게 됨.

2. 감사로그 성능 개선.

- 거래의 필요한 데이터들을 CUD시 이력을 JSON 형태로 남기고 있었음.

- before, after를 화면상에서 보여주고 서로의 변경된 부분을 하이라이트해서 보여주는 구조. (diff.js 사용.)

- JSON이 커지면 커질수록 before, after를 비교하는 diff.js에서 성능이 현저하게 느려져 화면이 멈추는 현상이 나타남.

- 이를 해결하고자 Web Worker를 사용함.

- WebWorker는 기본적으로 외부 js에 onmessage를 구현해야 하는데 유지 보수할 때 Vue 프로젝트와 Web Application에 있는 js를 번갈아 가면서 보면 가독성이 떨어지기 때문에 onmessage 를 아래와 같이 구현.

- new Worker(URL.createObjectURL(new Blob([ "onmessage를 구현한 function 소스(stringify)" ], { type: `application/javascript` })));

- onmessage를 구현한 function 소스(stringify) 에는 외부 라이브러리를 사용해야 하기 때문에 importScripts로 라이브러리를 로드하고 diff를 사용.

- 비교하는 로직 자체를 Web Worker를 사용 Background Thread로 돌려서 화면이 멈추는 현상을 해결 할 수 있었음.

3. 필드간의 Drag & Drop맵핑 기능

- iManager는 연계 시스템이다 보니 소스 시스템에서 타겟 시스템에 데이터를 전달할때 데이터 key가 다른 경우가 많음. ex) 소스 시스템에서는 사용자 이름을 userName 타겟 시스템에는 name.

- 기존에는 필드 간 맵핑을 eclipse 플러그인이 제공해주는 기능을 사용.

- 레거시한 eclipse 플러그인 대신 브라우저에서 해당 기능의 필요성이 높아져서 개발하게 됨.

4. LOWCODE화 시킨 사용자 정의 대시보드

- 기존의 구현했던 파워포인트와 같은 대시보드가 아닌 새로운 방식의 UI로 사용자가 원하는 위치에 각 성능 항목들을 배치 할 수 있는 기능 구현.

Graph 모듈 개발

그래프 알고리즘에서 사용되는 노드, 엣지등을 웹에서 직접 설정 할 수 있는 모듈 개발.
- 사용 기술
* JavaScript
- 인력 구성
* 개발자: 1명
- 상세 업무

연계 시스템 특성 상 개발에 시간이 걸리기 때문에 주로 사용되는 패턴을 node화 하고 node 간 진행 순서를 edge로 연결 해야 하는 모듈이 필요 했고 또한 타 솔루션 팀에서 도 비슷한 맥락으로 필요 했기 때문에 node를 drag&drop 하고 node간 연결 을 담당 하는 모듈을 개발함.

구현 예제

- 오른쪽 하단의 아이템1, 아이템2 탭의 항목들을 좌측의 모자이크 영역으로 drag & drop.

- 노드를 drag & drop 해서 위치를 이동.

- 모자이크 영역에 위치에 있는 각 노드들을 노드 상하좌우에있는 원을 drag & drop 해서 연결.

- 그룹 노드(아이템1 > Scope)를 모자이크 영역에 추가 하고 단일 노드들을 그룹 노드에 drag & drop.

- 그룹 노드 우측 하단을 drag 해서 크기를 수정.
