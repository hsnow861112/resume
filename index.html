Zenius APM (Web Application Server 모니터링)

Java를 기반으로 한 Web Application Server의 성능, 장애에 대한 모니터링 기능을 제공 하는 솔루션.
- 인력 구성
* 개발자: 4명 , 디자인: 1 명

- 초단위로 들어오는 성능 data를 웹 브라우저가 Memory Leak 가 발생 하지 않게 처리.

- WebWorker를 통한 UI 개선 경험.

- 대시보드의 구성을 사용자가 손쉽게 설정 할 수 있는 UX/UI 설계 및 구현.

- 상세 업무
1. 실시간 대시보드 개발

WAS에 인입되고 처리되는 모든 트랜잭션들을 실시간으로 모니터링하고, 지연 상황을 가시적으로 분석 할 수 있는 실시간 대시보드 개발.

2. X-VIEW 차트 개발

현재 WAS가 처리한 트랜잭션을 응답시간, 처리상태 별로 나누어서 표현하는 X-VIEW 차트를 html5 canvas를 사용해서 개발.

구현 예제

- 데이터 생성 및 삭제는 WebWorker를 통해 생성됨.

- 차트 드래그 시 선택 된 점의 개수를 차트 상단의 선택한 개수에서 확인 할 수 있음.

3. 스피드 바 차트 개발

RPS(초당 요청 건수), Active Transaction(현재 처리중인 건수), TPS(초당 처리 건수) 를 나타내는 차트를 html5 Canvas 및 애니메이션을 위한 반복함수 requestAnimationFrame을 기술 조사하여 차트를 개발함.

구현 예제

1. 상단의 스피드 바 차트 사진에서 왼쪽(초당 요청 건수), 오른쪽(초당 처리 건수) 영역의 애니메이션.

2. 실시간 요청이 서버(가운데 원통) 영역으로 들어 갔을때 버블 효과.

4. LOWCODE화 시킨 사용자 정의 대시보드

대시보드에 필요 하는 컴포넌트 항목을 drag & drop으로 사용자가 원하는 곳에 위치 시키고 컴포넌트들의 가로, 세로 크기도 drag 로 조절 가능하도록 설계 하고 이를 jQuery UI의 Draggable, Droppable, Resizable을 기술을 통하여 구현함.

Graph 모듈 개발

그래프 알고리즘에서 사용되는 노드, 엣지등을 웹에서 직접 설정 할 수 있는 모듈 개발.
- 인력 구성
* 개발자: 1명
- 주요 업무
Graph 모듈 개발
- 상세 업무

Source, Target 시스템간 data 연계 개발 시 자주 사용되는 코드 패턴들이 있는데 개발에 시간을 줄이고자 해당 코드 뭉치들을 그룹화(노드) 하고 해당 코드 뭉치들을 연결하게 해주는 Graph 모듈 개발.

구현 예제

- 사용법

- 오른쪽 하단의 아이템1, 아이템2 탭의 항목들을 좌측의 모자이크 영역으로 drag & drop.

- 노드를 drag & drop 해서 위치를 이동.

- 모자이크 영역에 위치에 있는 각 노드들을 노드 상하좌우에있는 원을 drag & drop 해서 연결.

- 그룹 노드(아이템1 > Scope)를 모자이크 영역에 추가 하고 단일 노드들을 그룹 노드에 drag & drop.

- 그룹 노드 우측 하단을 drag 해서 크기를 수정.
